HTTP and HTTPS

What is HTTP?

HTTP stands for Hypertext Transfer Protocol and it is the language used by computers to communicate over the World Wide Web. The Uniform Resource Locator (URL) is the identifier for a particular website, service, document, etc. Get requests are the standard method used by client computers to ask the web servers to provide a document.

What is HTTPS?

SSL stands for Secure Sockets Layer and TLS stands for Transport Layer Security. Both of these are protocols for communicating on a secure channel which is indicated by the HTTPS tag in the URL of the resource. A secure connection requires a digital certificate from the server to verify the authenticity of the web server. A Certificate Authority (CA) is the entity responsible for issuing the digital certificates and confirming the authenticity of web sites. A POST request is when the web browser sends data to the web server for authentication. If the credentials are authentic, the server responds with a login success message as well as cookie data. The cookie data is how the web server can easily identify you as the user you authenticated as. For future requests, the web browser includes the cookie data and the server responds quickly with the required data.

What is Encryption?

A URL that includes an HTTPS tag (or a lock icon) indicates a secure connection which requires encryption. Private data sent over the Internet (which is accessible to the public) can be kept secret using a process called encryption which alters the original message based on certain rules to hide the text. The rules for altering the text are contained within what is called an algorithm and the rules are based on a key which can be used to decode (decrypt) the message. Modern day encryption uses 256-bit keys which means each character in blocks of 256 characters in the message is altered differently. With the pattern only repeating for every 256 characters, it is very difficult for even modern day computers to decrypt but as time marches on and computer components increase in power we will need to increase the size of the keys used.

Types of Encryption

Symmetric Encryption is where, prior to sending any data, the sender and receiver agree upon the use of a single secret key for decrypting the private messages. Unfortunately, this process is not viable for a publicly accessible communication medium like the Internet. So computers use Asymmetric Encryption which includes a public key and a private key. Any computer can use a public key to send messages to particular end user but only the end user has the private key to access the messages and decrypt them. Asymmetric Encryption is the basis for all secure messaging on the Internet including the SSL and TLS protocols.
